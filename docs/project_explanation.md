# Project Explanation

## Title
End-to-End MLOps Project for Heart Disease Prediction

## Objective
The objective of this project is to build a complete machine learning pipeline for predicting heart disease using clinical features from the Cleveland heart disease dataset. The project demonstrates the full lifecycle of an ML system, from data ingestion to deployment.

## Workflow
1. Dataset is loaded from the attached archive.
2. Data validation and cleaning are performed.
3. Features are processed for model training.
4. Feature selection and hyperparameter tuning improve the model.
5. Model training and evaluation are completed.
6. Experiments are tracked with MLflow.
7. The trained model is served using FastAPI.
8. A Streamlit app provides a user-facing interface.
9. Docker and Kubernetes deployment files make the system portable.

## Key Technologies
- Python
- scikit-learn
- FastAPI
- Streamlit
- MLflow
- DVC
- Docker
- Kubernetes
- GitHub Actions
- Pytest

## Why This Project Is Important
This project shows how machine learning models can be built, tested, tracked, served, and deployed in a production-like workflow.
